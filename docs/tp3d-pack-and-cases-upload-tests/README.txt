Cargo-Rule V1 Sample Data
==========================

Files
-----
1. cargo_cases_valid.csv
2. cargo_cases_valid.xlsx
   Clean import data covering every Cargo-Rule V1 field and several rule combinations.

3. cargo_cases_warning_inputs.csv
4. cargo_cases_warning_inputs.xlsx
   Intentional invalid and extreme values for row-warning, fallback, and blocking-validation tests.

5. cargo_rule_test_matrix.xlsx
   Human QA checklist describing expected outcomes.

Import notes
------------
- CSV and XLSX valid files contain the same rows.
- Warning files contain the same rows.
- Import one file at a time.
- All case names in the valid file are unique.
- The final row in the warning file has a blank name and should be blocked.
- maxPalletWeight is warning-only, not an AutoPack hard limit.
- maxStackCount means direct children on a support; 0 means unlimited.
- noStackOnTop wins over maxStackCount but the stored maxStackCount should remain preserved.
- laneItem values test Automatic, Always, and Never.
- orientation aliases should all normalize to onSide.

Important
---------
These files test Case Library spreadsheet import and Cargo-Rule UI behavior.
They do not include Pack JSON, Workspace JSON, or App Backup JSON because the previously uploaded local files have expired and the exact current export envelope should be copied from a fresh export generated by the app at the current commit. A current exported Pack/App/Workspace sample can be used to build exact JSON fixtures without guessing the schema.
